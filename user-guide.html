Application Login & Admin Authentication

Welcome to the Braze to MoEngage Migration Tool! Let's begin with accessing the application and authenticating as an admin user.

What you'll see:

  • The application's main landing page with the migration tool interface
  • A clean, professional interface designed for easy navigation
  • Password input field for secure access
Important

You'll need a password to access the migration tool. Contact your system administrator to obtain the access password before proceeding.

Authentication Steps:

  1. Contact your system administrator to request the application access password
  2. Open the migration tool application in your web browser
  3. You'll be prompted to enter the password
  4. Enter the password provided by your admin
  5. Click "Enter" or "Login" to proceed to the main dashboard
Next Step

Once the password is validated and you're authenticated, you'll be redirected to the platform selection interface.

2

Platform Selection (Braze/Salesforce)

After successful authentication, you'll see the platform selection interface where you can choose which platform you want to migrate from. Currently, the tool focuses on Braze migration.

Available Migration Platforms:

  • Braze: Fully supported - Complete migration capabilities for campaigns and content blocks
  • Salesforce Marketing Cloud: Coming soon - Currently disabled/under development
Current Status

We are currently focusing on Braze to MoEngage migration. Salesforce integration is planned for future releases but is currently disabled.

Platform Selection Process:

  1. Review Platform Options: You'll see cards for available migration platforms
  2. Select Braze: Click on the Braze card (highlighted with a blue border when available)
  3. Note Salesforce Status: Salesforce will appear grayed out with "Disabled" status
  4. Proceed to Migration Type: After selecting Braze, you'll see migration type options

Migration Type Selection:

Once you select Braze as your platform, you'll see three migration type options:

  • Campaigns: Migrate your email, push, and SMS campaigns
  • Content Blocks: Migrate reusable content blocks and templates
  • Flows: Currently disabled - Workflow automation migration (coming soon)
Important

Make sure you have the necessary permissions in your Braze account to access campaigns and content blocks for the migration types you select.

Ready to Proceed

Select "Braze" as your platform and choose your migration type (Campaigns or Content Blocks) to continue to the authentication step.

3

Migration Type Selection

After selecting Braze as your platform, you need to choose what type of content you want to migrate from Braze to MoEngage.

Available Migration Types:

  • Campaigns: Fully supported
    • Email campaigns with templates and content
    • Push notifications for iOS, Android, and Web
    • SMS campaigns and messaging
    • User segmentation and targeting rules
    • Scheduling and automation settings
  • Content Blocks: Fully supported
    • Reusable email templates and headers
    • Rich media content and images
    • Dynamic content and personalization blocks
    • Footer templates and legal disclaimers
  • Flows: Coming soon - Currently disabled
    • Workflow automation and trigger campaigns
    • User journey mapping and sequences
    • Advanced behavioral targeting rules
Pro Tip

You can run separate migrations for Campaigns and Content Blocks. It's often recommended to migrate Content Blocks first, then Campaigns that depend on those blocks.

What Gets Migrated:

Campaign Migration Includes:
  • Email Campaigns: Subject lines, content, sender info, scheduling, A/B tests
  • Push Notifications: Titles, messages, images, action buttons, deep links
  • SMS Campaigns: Message content, sender information, delivery timing
  • Segmentation: Coming soon, under development
  • Analytics Setup: Coming soon, under development

Content Block Migration Details:

When migrating Content Blocks, the tool handles various types of reusable content components:

Content Block Types Supported:
  • HTML Content Blocks: Rich HTML templates with styling and formatting
  • Text Content Blocks: Plain text snippets and reusable copy
  • Image Content Blocks: Image assets with alt text and responsive settings
  • Dynamic Content Blocks: Personalized content with liquid templating
  • Conditional Logic Blocks: Content that changes based on user attributes
  • Footer Templates: Email footers with unsubscribe links and legal text
  • Header Templates: Branded headers with logos and navigation
Migration Notes
  • Content blocks are migrated first to ensure campaigns that reference them work correctly
  • Liquid templating syntax is automatically converted to MoEngage's personalization format
  • Image assets are transferred and hosted on MoEngage's CDN
  • Any custom CSS or styling is preserved during migration
Ready to Proceed

Select your desired migration type (Campaigns or Content Blocks) and click "Continue" to proceed to the Braze authentication step.

4

Braze Authentication & API Setup

Now you'll need to authenticate with your Braze account to allow the migration tool to access your campaigns and data.

Watch the complete guide for setting up Braze credentials

Required Braze Credentials:

  1. API Key: Your Braze REST API key with appropriate permissions
  2. App Identifier: The unique identifier for your Braze app
  3. Instance URL: Your Braze instance endpoint (e.g., dashboard-01.braze.com)
  4. Workspace ID: The ID of the Braze workspace you're migrating from

Step-by-Step Authentication:

  1. Access Braze Dashboard:
    • Log into your Braze account
    • Navigate to Settings → API Keys
  2. Create or Select API Key:
    • Create a new API key or use an existing one
    • Ensure it has the following permissions:
      • campaigns.list • campaigns.details • content_blocks.list • content_blocks.details • segments.list • segments.details • users.export.ids
  3. Copy Required Information:
    • Copy your API key
    • Note your instance URL
    • Get your App Identifier from Settings → App Settings
  4. Enter Credentials in Migration Tool:
    • Paste the API key in the designated field
    • Enter your instance URL
    • Input your App Identifier and Workspace ID
Security Note

Keep your API credentials secure and never share them publicly. The migration tool uses secure connections and doesn't store your credentials permanently.

Troubleshooting

If authentication fails, verify that:

  • Your API key has the correct permissions
  • The instance URL is correct and accessible
  • Your Braze account is active and in good standing
  • There are no IP restrictions blocking the migration tool
5

MoEngage Credentials Setup & Authentication

The final authentication step involves setting up your MoEngage account credentials to enable the migration tool to create campaigns in your MoEngage workspace.

Complete video guide for MoEngage credentials setup

Required MoEngage Credentials for Content Block Migration:

MoEngage uses Basic Authentication for API requests. You'll need to gather the following credentials from your MoEngage dashboard:

  1. Workspace ID (Username): Your MoEngage workspace identifier (formerly App ID)
  2. API Key (Password): Your MoEngage API key for authentication
  3. Data Center: Your MoEngage data center region (US, EU, etc.)
Authentication Method

MoEngage APIs use Basic Authentication with Base64 encoding. The migration tool will automatically create the required authorization header in this format:

{"Authorization":"Basic Base64_ENCODED_WORKSPACEID_APIKEY=="}

The API request will be authenticated through Basic Authentication. Basic Authentication sends a Base64-encoded string containing your username and password with every API request. It encodes a 'username:password' string in Base64 and appends the encoded string with 'Basic '.

MoEngage Workspace API configuration interface

How to Get Your MoEngage Credentials:

Follow these detailed steps to obtain your MoEngage credentials from the MoEngage dashboard:

Step 1: Access MoEngage API Settings

  1. Log in to your MoEngage dashboard
  2. On the left navigation menu, go to Settings > Account > APIs

Step 2: Copy Workspace ID (Username)

  1. In the APIs section, locate the Workspace ID (earlier called App ID)
  2. Click the copy icon next to the Workspace ID to copy the username
  3. This will serve as your username for Basic Authentication

Step 3: Copy API Key (Password)

  1. In the API keys section, find the Push templates tile
  2. Click the copy icon in the Push templates tile to copy the API key
  3. This will serve as your password for Basic Authentication

All required fields for MoEngage content block migration setup

Step 4: Identify Your Data Center

  1. Check your MoEngage dashboard URL to identify your data center:
    • US: dashboard-01.moengage.com
    • EU: dashboard-02.moengage.com
    • APAC: dashboard-03.moengage.com
Important Security Notes
  • Keep your API credentials secure and never share them publicly
  • The API key provides access to your MoEngage workspace, treat it like a password
  • Use these exact details to authenticate API requests
  • Ensure your API key has campaign creation and content block management permissions

Enter Credentials in Migration Tool:

  1. Workspace ID: Paste the Workspace ID you copied from Settings > Account > APIs
  2. API Key: Enter the API Key from the Push templates tile
  3. Data Center: Select your data center region based on your dashboard URL
  4. Test Connection: Click "Test Connection" to verify the credentials
  5. Save Configuration: Once verified, click "Save Credentials"
Content Block Migration Setup

For content block migrations, ensure you have the following additional permissions:

  • Content block creation and management
  • Template upload and modification rights
  • Asset management for images and media
Connection Verified

When you see the "Connection Successful" message, your MoEngage credentials are properly configured and you're ready to begin the migration process.

Important Notes
  • Double-check your data center selection to avoid connection issues
  • Ensure your MoEngage account has sufficient quota for the campaigns you're migrating
  • Keep your API credentials secure and rotate them periodically
  • For content block migrations, verify that your workspace has content block creation permissions enabled
6

Migration Process & Completion

Now that both Braze and MoEngage credentials are configured, you can begin the actual migration process. The tool will automatically handle the transfer of your selected campaigns and content.

Migration Process Overview:

  1. Pre-Migration Validation:
    • The tool validates all selected campaigns
    • Checks for compatibility issues
    • Identifies any missing content or dependencies
  2. Content Block Migration:
    • Migrates reusable content blocks first
    • Preserves formatting and dynamic content
    • Maps personalization variables to MoEngage format
  3. Campaign Migration:
    • Transfers campaign configurations
    • Preserves scheduling and segmentation rules
    • Converts trigger conditions and automation logic
  4. Testing & Validation:
    • Runs automated tests on migrated campaigns
    • Validates content rendering
    • Checks personalization accuracy

Migration Steps:

  1. Review Selection:
    • Confirm the campaigns and content blocks selected for migration
    • Review any warnings or compatibility notes
  2. Start Migration:
    • Click "Begin Migration" to start the process
    • Monitor the progress bar and status updates
  3. Monitor Progress:
    • Track the migration status in real-time
    • View detailed logs for each campaign
    • Address any errors or warnings as they appear
  4. Review Results:
    • Check the migration summary report
    • Verify that all campaigns were migrated successfully
    • Note any manual adjustments needed
What Gets Migrated
  • Email Campaigns: Subject lines, content, sender info, scheduling
  • Push Notifications: Titles, messages, images, action buttons
  • SMS Campaigns: Message content, sender information, timing
  • Segmentation: Audience definitions and targeting criteria
  • Automation: Trigger conditions and workflow logic

Post-Migration Checklist:

  1. Campaign Review:
    • Log into your MoEngage dashboard
    • Review each migrated campaign
    • Test email rendering and personalization
  2. Segmentation Verification:
    • Verify that audience segments were created correctly
    • Check segment criteria and size estimates
  3. Content Validation:
    • Test all dynamic content and personalization
    • Preview campaigns across different devices
    • Verify link tracking and UTM parameters
  4. Schedule Setup:
    • Review and adjust campaign schedules as needed
    • Set up any additional automation workflows
Migration Complete!

Congratulations! Your campaigns have been successfully migrated from Braze to MoEngage. You can now manage and optimize your campaigns in the MoEngage platform.

Next Steps
  • Conduct thorough testing before activating migrated campaigns
  • Update any hardcoded links or references to Braze
  • Train your team on MoEngage's interface and features
  • Set up monitoring and analytics to track campaign performance
